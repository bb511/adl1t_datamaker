# Prescale Files

These are stored in the `L1Menus` folder.
The menu files are prescale files that determine the rate of certain objects as they are recorded by the trigger.
For example, `L1_SigleMuOpen` might trigger all the time, and hence only one in `n` values are actually recorded, where `n=63000` for the `Prescale_2022_v0_1_1.csv` menu.
They are used in the current code to get the relevant seeds, i.e., which triggers were operational, for a certain set of runs.

# Pileup Files

These are stored in the folder `pileup_files`.
Files with pileup information generated by brilcalc for all the runs that have been processed to date.
If processing a new run and want to include pileup information in the final h5, you need to add a `brilcalc` generated file with pileup information to the pileup folder here and respect the naming convention: `run[run_number]_brilcalc_PU`.
